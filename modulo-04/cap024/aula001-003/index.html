Trabalhando com Formulários

O básico de todo formulário é, primeiramente, estar envolvido nas tags <form> e </form>.

Para começar, um dos elementos mais importantes é o <input>. Ele serve para coletar os dados do usuário através de uma caixa na qual pode ser digitada. Um input pode ser de inúmeros tipos, cada tipo tem seus estilos predefinidos para facilitar a coleta dos dados (text, password, email, datetime, submit, etc).

Abaixo estamos usando dois tipos de input: um input text para a coleta do nome e sobrenome do usuário, e um input submit para enviar os dados escritos no input text para algum fluxo de tratamento.

A partir de inputs do tipo submit, podemos enviar dados diretamente para arquivos de linguagens back-end, como PHP, que vão tratar os dados enviados.

Diferença entre "name" e "id"

Ambos são identificadores, sua diferença está para que estão direcionados. Quando enviamos dados via um submit, geralmente mandamos esses dados para alguma linguagem de programação back-end para tratá-los. O "name" nesse caso, é útil justamente para isso, na linguagem PHP, por exemplo, a identificação dos dados é feita a partir de "names", o que facilita sua manipulação, enquanto a linguagem JavaScript trabalha melhor com os IDs. Além disso, IDs são usados para linkar inputs e suas respectivas labels.

Utilizando Labels

Labels são tags usadas para envelopar os identificadores textuais dos inputs. No exemplo acima, "Nome" e "Sobrenome" deveriam estar dentro de labels. Sua principal utilidade é criar uma relação entre o texto e o input, melhorando o SEO de seu site. Além disso, ele cria uma relação na usabilidade do formulário, uma vez que, ao clicar sobre uma label, você seleciona o input relacionado a ela.

Métodos de Envio de Formulários

Quando clicamos num submit para enviar os dados para outro lugar, a HTML pode fazer essa transferência de 2 maneiras. Essas maneiras são chamados de Métodos de Envio, e elas são: get e post. Get é o método menos seguro dos dois, uma vez que ele envia os dados explicitamente através da URL. Post, por sua vez, não envia diretamente pela URL, mas ainda assim deixa os dados acessíveis através da área "Payload" do arquivo de destino, visível pelo DevTools do Chrome.

Quando usar cada método?

Para decidir qual método usar, podemos nos basear em alguns fatores: usamos o GET majoritariamente quando:

  • Dados passados não são sensíveis;
  • Seus dados não ultrapassarem de 3 mil bytes (aproximadamente 3 mil letras);
  • Seus dados forem apenas textuais e não mídias como fotos/vídeos;
  • Você desejar ter uma URL com dados compartilhável.

Já em relação ao método POST, usamos quando estamos tratando de:

  • Dados sensíveis;
  • Dados maiores que 3 mil bytes;
  • Dados não textuais;
  • URLs não compartilháveis.

Vale lembrar que o uso do método POST para dados sensíveis como senhas ou dados de cartão de crédito não é o mais ideal. Em vez disso, devemos usar o protocolo HTTPS para trafegar com esse tipo de informação.
